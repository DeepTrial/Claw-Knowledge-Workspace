# 新 Agent 快速上手指南

> 欢迎来到 ZIV-BOT 知识库！预计阅读时间：5 分钟

---

## 🎯 知识库用途

本知识库存储深度调研成果，供 Agent 学习和复用：

- **课题调研** → `TOPICS/` 目录
- **技能文档** → `SKILLS/` 目录  
- **最佳实践** → `BEST_PRACTICES/` 目录

---

## 🚀 5 分钟快速上手

### Step 1: 了解知识地图（1 分钟）

阅读 `INDEX.md`，了解知识库全貌。

### Step 2: 学习格式规范（2 分钟）

阅读 `FORMAT.md`，了解知识卡片格式。

### Step 3: 按需查阅（2 分钟）

根据任务需求，查阅 `TOPICS/` 或 `SKILLS/` 中的具体知识卡片。

---

## 📖 知识卡片阅读顺序

每个知识卡片按以下顺序阅读效率最高：

```
1. 核心结论 ─→ 一句话理解
2. 关键要点 ─→ 3-5 个要点
3. 深度内容 ─→ 详细调研成果
4. 相关资源 ─→ 延伸阅读
```

## ✍️ 贡献知识

完成深度调研后，按以下步骤贡献知识：

### 1. 创建知识卡片

```bash
# 选择目录
TOPICS/       # 课题调研
SKILLS/       # 技能文档
BEST_PRACTICES/  # 最佳实践

# 按 FORMAT.md 格式创建
touch TOPICS/topic-xxx.md
```

### 2. 更新索引

更新 `INDEX.md` 中对应的索引表格。

### 3. 通知其他会话

```
# 使用 sessions_send 分享重要发现
sessions_send(sessionKey="xxx", message="新知识: KB-xxx")
```

## ⚠️ 注意事项

1. **避免重复调研** - 先搜索 `INDEX.md` 或使用 `memory_search`
2. **保持格式一致** - 严格遵循 `FORMAT.md` 规范
3. **及时更新索引** - 新增知识后更新 `INDEX.md`
4. **标注知识状态** - 使用正确的状态标签

---

## 🆘 常见问题

### Q: 如何判断知识是否已存在？

A: 先查看 `INDEX.md` 索引，或使用 `memory_search` 搜索关键词。

### Q: 知识卡片太长怎么办？

A: 确保核心结论和关键要点简洁，详细内容放深度内容部分。

### Q: 如何更新已过时的知识？

A: 更新知识卡片内容，修改状态为 `#deprecated` 或直接更新后标注更新时间。
