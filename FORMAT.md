## 📝 字段说明

### 必填字段

| 字段 | 说明 | 示例 |
|------|------|------|
| ID | 唯一标识 | `KB-20260305-001` |
| 核心结论 | 一句话总结 | 必填，其他 Agent 快速理解 |
| 关键要点 | 3-5 个要点 | 列表形式 |
| 标签 | 分类标签 | `#AI #LLM` |

### 可选字段

| 字段 | 说明 |
|------|------|
| 领域 | 技术领域分类 |
| 难度 | beginner/intermediate/advanced |
| 状态 | draft/reviewed/mastered |
| 前置知识 | 学习此知识前需要了解的 |
| 深度内容 | 详细调研成果 |
| 相关资源 | 外部链接 |
| 知识关联 | 内部链接 |

---

## 🏷️ ID 命名规范

| 类型 | 前缀 | 格式 | 示例 |
|------|------|------|------|
| 课题调研 | KB | `KB-YYYYMMDD-XXX` | `KB-20260305-001` |
| 技能文档 | SKILL | `SKILL-YYYYMMDD-XXX` | `SKILL-20260305-001` |
| 最佳实践 | BP | `BP-YYYYMMDD-XXX` | `BP-20260305-001` |

---

## 🏷️ 标签规范

### 领域标签

- `#AI` - 人工智能
- `#LLM` - 大语言模型
- `#DevOps` - 运维开发
- `#Frontend` - 前端开发
- `#Backend` - 后端开发
- `#Database` - 数据库
- `#Security` - 安全

### 状态标签

- `#draft` - 草稿
- `#reviewed` - 已审核
- `#mastered` - 已掌握
- `#deprecated` - 已过时

### 优先级标签

- `#critical` - 关键知识
- `#important` - 重要知识
- `#reference` - 参考知识

---

## 📂 文件命名规范

| 目录 | 文件名格式 | 示例 |
|------|------------|------|
| TOPICS/ | `topic-xxx.md` | `topic-rust-async.md` |
| SKILLS/ | `skill-xxx.md` | `skill-clawhub.md` |
| BEST_PRACTICES/ | `practice-xxx.md` | `practice-git-commit.md` |

---

## ✅ 质量检查清单

创建知识卡片前，确认：

- [ ] ID 唯一且符合规范
- [ ] 核心结论清晰（一句话能理解）
- [ ] 关键要点 3-5 个
- [ ] 标签正确
- [ ] 已更新 INDEX.md 索引
